0. 세팅

1) Intermediate : 빌드 산출물들

2) 언리얼은 기본적으로 에디터에서 플러그인들을 DLL 형태로 가져다 쓰고,
패키징된 exe 파일은 스태틱 라이브러리를 가져다 쓴다.

3) UnrealInsights : 언리얼엔진 내에서 시각적으로 성능 사용량을 확인할 수 있다.

4) UnrealHeaderTool : 사용자가 작성한 코드들을 generated header로 만들어주는 툴

5) ShaderCompileWorker : 언리얼 Shader를 병렬처리하는 툴
ㄴ .ush : 언리얼 Shader 확장자. HLSL로 작성됨

6) Source
ㄴ Developer : 개발중인 컨텐츠
ㄴ Editor : 에디터에서만 사용해야되는 컨텐츠 (예외처리가 필요하다)
    에디터 내에서 자동화 툴을 만들기도 한다.
ㄴ Runtime : Core, Engine, Actor 등등등 에디터 여부와 상관없이 사용하는 코드들
ㄴ Thirdparty : 외부 라이브러리들



1. Sharpmake 
유비소프트에서 CMake(빌드 자동화 툴)을 대체하기 위해 만든 라이브러리
C#으로 작성된 코드를, batch 스크립트로 실행
CMake 설정 파일을 통해 binary 파일을 읽고, 솔루션을 만들 수 있다.
폴더 내에서 우클릭 -> 추가 옵션 -> Visual Studio로 열기 로 솔루션을 열 수 있다.

EngineMain.build.cs를 시작으로 경로를 바탕으로 다른 빌드 스크립트들과 유틸리티 스크립트를 추가시키고 있다.
AddModule로 다른 cs 파일들을 추가해서 실행시키고 있다.
빌드 옵션, 
솔루션에 .cs .ini .hlsl 추가, 
솔루션 이름 설정, 
빌드 구성 이름 변경 (Release -> Develop)
유니코드, 
점보 빌드,
C++ 20으로 버전 설정,
인라인 등 디버그 설정,
핫리로드 대응,
RTTI 설정,
OutputPath, IntermediatePath, TargetPath등을 설정,
솔루션 파일 위치 설정 
실행할 파일 명 설정
실행시 빌드 스크립트를 통해 변경 사항, 유저 설정들을 복사해 Binary 파일에 추가
devenv를 통해 실제 프로젝트를 빌드하는 것처럼 만들기
등을 하고 있다.

CommonProject를 상속한 스크립트 등을 실행함으로써, 프로젝트를 생성하고 있다.
엔진과 유사한 프로젝트 구성을 위해, 실제 빌드할 프로젝트들은 Intermediate에 있고, 
CustumBuild 명령어를 사용하여 한 프로젝트안에 구성되도록 만들었다.

순서가 꼬이지 않도록, Super를 타고 들어가 부모부터 로드

   - Actor 생성
     ㄴ Template ? Template : CDO 사용하여 생성
     ㄴ NewObject 내에서 UPROPERTY를 복제

     ㄴ PostActorConstruction : Blueprint Construction Script 실행 (본 프로젝트에서는 다루지 않음)

3. Launch
