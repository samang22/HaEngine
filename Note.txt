0. 세팅

1) Intermediate : 빌드 산출물들

2) 언리얼은 기본적으로 에디터에서 플러그인들을 DLL 형태로 가져다 쓰고,
패키징된 exe 파일은 스태틱 라이브러리를 가져다 쓴다.

3) UnrealInsights : 언리얼엔진 내에서 시각적으로 성능 사용량을 확인할 수 있다.

4) UnrealHeaderTool : 사용자가 작성한 코드들을 generated header로 만들어주는 툴

5) ShaderCompileWorker : 언리얼 Shader를 병렬처리하는 툴
ㄴ .ush : 언리얼 Shader 확장자. HLSL로 작성됨

6) Source
ㄴ Developer : 개발중인 컨텐츠
ㄴ Editor : 에디터에서만 사용해야되는 컨텐츠 (예외처리가 필요하다)
    에디터 내에서 자동화 툴을 만들기도 한다.
ㄴ Runtime : Core, Engine, Actor 등등등 에디터 여부와 상관없이 사용하는 코드들
ㄴ Thirdparty : 외부 라이브러리들



1. Sharpmake 
유비소프트에서 CMake(빌드 자동화 툴)을 대체하기 위해 만든 라이브러리
C#으로 작성된 코드를, batch 스크립트로 실행
CMake 설정 파일을 통해 binary 파일을 읽고, 솔루션을 만들 수 있다.
폴더 내에서 우클릭 -> 추가 옵션 -> Visual Studio로 열기 로 솔루션을 열 수 있다.
